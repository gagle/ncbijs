# scripts — workspace utility scripts

Auto-loaded when working anywhere in `scripts/`.

## Purpose

Repo-internal automation. **Not published.** Three jobs live here today:

- `sync-docs.ts` — regenerates routing tables in root `README.md` and `CLAUDE.md` from the `docs-manifest/` manifests + per-package CLAUDE.md frontmatter.
- `check-claude-md.ts` — runs sustainability checks (token budgets, freshness, export drift, link validity, dep-graph symmetry, pitfall surfacing) over every CLAUDE.md in the workspace.
- `ncbi-api-monitor/` — weekly drift detector for upstream NCBI APIs. See [`ncbi-api-monitor/CLAUDE.md`](./ncbi-api-monitor/CLAUDE.md) for its detection strategy.

## Project layout

```
    detect.spec.ts
    CLAUDE.md
```

One Nx project, `ncbijs-scripts`. Configs at `scripts/` root cover every `**/*.ts` underneath.

## Adding a new script

1. Drop `<name>.ts` and `<name>.spec.ts` at `scripts/` root (or under a subdirectory if it's a multi-file tool).
2. Export pure helpers; guard the entry point so the spec can import without triggering `main()`:
   ```ts
   const entryPath = process.argv[1];
   if (entryPath !== undefined && import.meta.url === `file://${entryPath}`) {
     main();
   }
   ```
3. If the script is invoked frequently, add a `pnpm <name>` shortcut in root `package.json`.
4. Run `pnpm nx run ncbijs-scripts:lint` and `:test`. Both are wired into `/verify`.

## Testing conventions

- 100% coverage target via `restoreMocks: true` and v8 (matches the broader testing rules).
- Pure helpers: test directly with hardcoded inputs.
- Filesystem-touching helpers: use `mkdtempSync(tmpdir())` + `writeFileSync` to build a fixture tree per test, clean up in `afterEach`.
- Don't mock the workspace's real files — use temp dirs.

## Common pitfalls

1. **Importing `main()`-side-effects.** Forgetting the entry guard makes the spec invoke the script. Symptom: tests blow up at import time. Fix: wrap `main()` invocation in the `import.meta.url === file://process.argv[1]` check.

2. **`require`-style paths.** ESM only — use `node:url` + `fileURLToPath` for `__filename` / `__dirname`. The script is parsed under `tsconfig.base.json` which enforces `moduleResolution: bundler`.

3. **Adding NCBI-specific code to root scripts.** `sync-docs.ts` and `check-claude-md.ts` are workspace tools and must stay project-agnostic-adjacent. Anything NCBI-specific belongs in `scripts/ncbi-api-monitor/` (or a new subdirectory).

4. **Editing the generated table regions in root README/CLAUDE.md.** Anything between `<!-- sync-docs:<region>:start -->` and `:end` is regenerated by `pnpm sync-docs`. Edit the manifest or per-package CLAUDE.md frontmatter instead.

## Cross-cutting wiring

- **`/verify` skill** runs `pnpm sync-docs --check` + `pnpm check-claude-md budget` + `pnpm check-claude-md freshness` (deterministic gates).
- **`pnpm sync-docs`** reads `docs-manifest/workflows.ts`, `docs-manifest/decision-tree.ts`, and every `packages/*/CLAUDE.md` frontmatter.
- **`scripts/ncbi-api-monitor/detect.ts`** is invoked by `.github/workflows/ncbi-api-monitor.yml` and the `/ncbi-check-updates` skill.
