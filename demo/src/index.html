How It Works

1
NCBI Servers
MeSH, ClinVar, Gene, PMC bulk datasets hosted on NCBI FTP
2
ncbijs HTTP APIs
@ncbijs/* packages fetch and parse real biomedical data
3
DuckDB File
@ncbijs/store writes records into a 4 MB .duckdb file
4
Browser DuckDB
DuckDB-Wasm loads the file and runs SQL queries client-side

The Live API panel queries NCBI servers in real time using @ncbijs/mesh, @ncbijs/datasets, @ncbijs/clinvar, @ncbijs/pubchem, and @ncbijs/id-converter.

The Local DuckDB panel queries a pre-built database containing real NCBI data: 188 MeSH descriptors, 20 genes, 60 ClinVar variants, 15 PubChem compounds, and 11 ID mappings. The database was built using @ncbijs/store's DuckDbFileStorage and is served as a static 4 MB file.

Both panels run the same query simultaneously so you can compare live API responses with locally stored data side by side.
